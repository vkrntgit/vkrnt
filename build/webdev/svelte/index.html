Svelte + Svelte Kit notes

I picked Svelte Kit in Jan 2022. Great framework to build node apps. I liked it so much that I moved this site from plain html+css site to Sveltei Kit. The framework is SEO friendly, pre-renders pages and allows rapid development of components and static pages.


Here are a few basic instructions to get started on Svelte Kit

How to start a Svelte project
npm init svelte@next

How to build and deploy
npm install
npm run build
npm run dev&
(to run npm in backgroud)

In case you run without ‘&’ (in foreground), use right Ctrl and ‘c’ to terminate server and return to terminal prompt

Copy and push public folder to deploy to production



Save layout to src/routes/__layout.svelte This file should have the header, main and footer tags with elements common to all pages. Use < slot >   < /slot > for content specific to each page. In the individual pages, you don’t need to code the common elements. Jump directly to components and text specific to that page.  Save css to src/styles/global.css

Containarize your sveltkit site


Create a docker image of svelte kit code.

Steps:
1. Open package.json. Change adapter-auto to adapter-node
2. Open package-lock.json. Change adapter-auto to adapter-node
3. npm run build
4. node build
5. Create Dockerfile
6. Enter
FROM node:16
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm build && npm prune --production
ENV PORT 5050
EXPOSE 5050
CMD ["node","build"]
7. Create .dockerignore
8. Enter these files
.git
.svelte-kit
build
node_modules
.dockerignore
.gitignore
Dockerfile
README.md
